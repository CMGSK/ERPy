# ERPy

ERPy is a lightweight, open inexpensive and simplified ERP for very small business to manage their resources. It is also an individual ongoing project, so beware of the stage of development, possible failures, and long periods of inactivity.

## Status: Early stage development
There are some major TO-DOs in this project. I will try to keep a track of the state on this file, so anyone who wants to contribute is able to do so, and for personal tracking too. 

#### Current milestones:
- Define the final database structure
- Redesign the main page structure
- Redesign tables
- Add filters to tables and combos
- Test functionalities
- Dynamically create .env files on pop-up configuration windows
- Create an autentication system
 

## Installation

ERPy currently runs under Python 3.11.

Download the zip (not available yet) or clone the repository in your local machine, and generate an access to the main.py file within the frontend package to run.

Firstly, you will need to have all dependencies, which can be installed with ```pip```. I highly recommend using a virtual environment while the first stages of the project are ongoing, or if you plan to contribute to the development.

```bash
pip install -r requirements.txt
```

## Contributing

Pull requests are welcome. For major changes, please open an issue first
to discuss what you would like to change.

Please take into consideration explaining me your changes, since this is a personal project and I'm on a learning journey (I've been programming in python for little time at the time this is written)

Please make sure to thoroughly test your contributions avoiding changing working behaviors, heavy GUI changes, or crashes.

Finally, please have in mind this is an OVERSIMPLIFIED ERP for really small business. I do no intend to create a full fledged ERP, but to develop a tool to ease people's work whom doesn't want to pay for one or go through the hassle of learning the ropes on one of the free alternatives.
